Glossary

This glossary provides definitions for key terms and concepts used throughout the Physical AI and Humanoid Robotics textbook.

A​

Action (in ROS 2): A communication pattern for long-running tasks that provide feedback during execution and can be preempted.

AI (Artificial Intelligence): The simulation of human intelligence processes by machines, especially computer systems.

Autonomous System: A system that operates independently without human intervention, making decisions based on sensor input and pre-programmed logic.

B​

Balance Control: The ability of a humanoid robot to maintain its center of mass within its support polygon to prevent falling.

Behavior Tree: A hierarchical structure used in robotics and AI to organize and execute complex behaviors.

C​

Computer Vision: A field of artificial intelligence that trains computers to interpret and understand the visual world.

Control System: A system that manages, commands, directs, or regulates the behavior of other devices or systems.

Cyber-Physical System: A system of collaborating computational elements controlling physical entities.

D​

DDS (Data Distribution Service): A specification that enables scalable, real-time, dependable, high-performance, and interoperable data exchanges.

Deep Learning: A subset of machine learning based on artificial neural networks with representation learning.

Digital Twin: A virtual representation of a physical system that simulates its behavior and characteristics.

E​

Embodied AI: Artificial intelligence that is integrated with a physical form and interacts with the physical world.

Environment Mapping: The process of creating a representation of the environment for navigation and planning.

F​

Forward Kinematics: The use of joint parameters to compute the position and orientation of the end-effector.

G​

Gazebo: A 3D dynamic simulator with accurate physics simulation and rendering capabilities used for robotics applications.

H​

Humanoid Robot: A robot with physical characteristics that resemble those of a human.

Human-Robot Interaction (HRI): The field of study dedicated to understanding, designing, and evaluating robotic systems for human use.

I​

Inverse Kinematics: The mathematical process of calculating joint parameters needed to place the end-effector at a specific position and orientation.

Isaac Sim: NVIDIA's simulation environment for developing AI-powered robots with photorealistic rendering capabilities.

K​

Kinematics: The study of motion without considering the forces that cause the motion.

L​

Large Language Model (LLM): A language model with many parameters that is trained on large amounts of text data.

LiDAR (Light Detection and Ranging): A remote sensing method that uses light in the form of a pulsed laser to measure distances.

Locomotion: The ability to move from one place to another, particularly in humanoid robots.

M​

Machine Learning: A method of data analysis that automates analytical model building using algorithms that iteratively learn from data.

Manipulation: The ability of a robot to interact with and control objects in its environment.

Middleware: Software that provides common services and capabilities to applications beyond what's offered by the operating system.

N​

Nav2: The ROS 2 navigation stack that provides path planning, obstacle avoidance, and localization capabilities.

Node (in ROS): A process that performs computation in the ROS system.

P​

Perception: The ability of a robot to interpret sensory information from its environment.

Physics Engine: Software that simulates the behavior of objects in a realistic way, including collision detection and response.

Point Cloud: A set of data points in space, typically representing the external surface of an object captured by 3D scanning.

Q​

QoS (Quality of Service): Settings in ROS 2 that specify how data is delivered between nodes, including reliability and durability parameters.

R​

Robot Operating System (ROS): A flexible framework for writing robot software that provides a collection of tools, libraries, and conventions.

ROS 2: The second generation of the Robot Operating System with improved architecture for real-world deployment.

Rigid Body Dynamics: The simulation of the motion of rigid bodies under the influence of forces and torques.

S​

Sensor Fusion: The process of combining data from multiple sensors to improve the accuracy and reliability of information.

Service (in ROS): A synchronous request/response communication pattern between nodes.

Simulation-to-Reality Transfer: The process of transferring skills or behaviors learned in simulation to real-world robotic systems.

SLAM (Simultaneous Localization and Mapping): The computational problem of constructing or updating a map of an unknown environment while simultaneously keeping track of an agent's location within it.

State Estimation: The process of determining the state of a system from noisy and incomplete measurements.

T​

Topic (in ROS): A communication channel used for asynchronous message passing between nodes in a publish/subscribe pattern.

U​

URDF (Unified Robot Description Format): An XML format for representing a robot model including kinematic and dynamic properties.

V​

VLA (Vision-Language-Action): Systems that integrate visual perception, language understanding, and physical action capabilities.

VSLAM (Visual Simultaneous Localization and Mapping): SLAM using visual sensors as the primary source of information.

W​

Whisper: OpenAI's automatic speech recognition system used for voice-to-text conversion.

X, Y, Z​

XYZ: Common notation for 3D coordinate systems representing the three spatial dimensions.

This glossary provides quick reference definitions for the technical terminology used throughout the textbook. For more detailed explanations of concepts, refer to the relevant chapters in each module.
